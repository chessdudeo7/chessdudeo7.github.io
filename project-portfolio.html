Overview

This is the site you're browsing right now. I built it entirely from scratch — no Bootstrap, no Tailwind, no component libraries. Every layout, animation, and interaction is hand-coded with vanilla HTML, CSS, and JavaScript.

The goal was to create a professional online presence that reflects my aesthetic sensibility and technical abilities simultaneously, while being fast, accessible, and easy to maintain.

How It Was Built

1

Design System First

Started with style.css — defining all CSS variables for color, spacing, and typography before writing a single component. This made the whole site feel consistent from the start.

2

Layout & Components

Built reusable components — navbar, hero, project cards, buttons — as pure HTML/CSS classes that could be dropped into any page without duplication.

3

Animations

Used IntersectionObserver for scroll-triggered reveals, CSS keyframe animations for the typing effect and floating icons, and a canvas animation loop for background particles.

4

Firebase Integration

Added Firestore for the tutoring review system — users submit star ratings and text reviews that persist across sessions and load in real time for all visitors.

5

Contact & Deployment

Integrated EmailJS to handle the contact form server-side-free, then deployed the site and iterated on layout and performance based on testing across devices and browsers.

Challenges

No Framework, No Shortcuts

Without a CSS framework, every layout, every hover state, and every responsive breakpoint had to be written by hand. Time-consuming, but it meant total control over the output and a much deeper understanding of CSS.

Cross-Page Consistency

Keeping the visual language consistent across Profile, Tutoring, and Projects pages — without a component system — required disciplined reuse of shared CSS classes and careful attention to detail.

Canvas Performance

The animated particle backgrounds needed to stay smooth at 60fps without causing jank during scroll. Tuned particle count and draw logic to keep the animation lightweight.

What I Learned

Building this site from scratch was one of the most educational frontend experiences I've had. Writing every CSS rule by hand gave me a level of fluency with layout, specificity, and the cascade that I couldn't have gotten from using a framework. I also got comfortable with the Firebase ecosystem and learned how to design something that's both technically clean and visually polished — a balance I'll carry into every future project.
