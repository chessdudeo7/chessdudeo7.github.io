Overview

KnightMare is an AI chess engine I built for ChessHacks, a chess-themed hackathon. The goal was to train a neural network that could look at any chess position and output a move quality score and a recommended best move — essentially replicating the "thinking" of a strong player using deep learning.

Rather than using traditional alpha-beta search like Stockfish, KnightMare learns patterns directly from games played by grandmasters, letting it develop an intuitive positional sense rather than brute-force calculation.

How It Was Built

1

Data Collection & Parsing

Downloaded a large PGN dataset of grandmaster games. Parsed each game move-by-move using python-chess, converting board states into numerical tensor representations the model could learn from.

2

Board Encoding

Encoded each position as a multi-channel binary tensor (12 channels — one per piece type per color, plus auxiliary channels for castling rights and en passant), giving the model a full spatial picture of the board.

3

Model Architecture

Designed a convolutional neural network in PyTorch. Convolutional layers extract spatial piece patterns; fully-connected layers produce a move probability distribution over all legal moves.

4

Cloud Training with Modal

Used Modal to spin up GPU instances for training — cutting what would have been hours of local CPU compute down to minutes. Iterated over hyperparameters (learning rate, batch size, network depth) across multiple runs.

5

Inference & Testing

Wrapped the trained model in a simple Python interface so it could accept a FEN string and return a top-3 move ranking. Tested against known puzzles and positions to validate move quality.

Challenges

Move Legality

Early versions of the model occasionally suggested illegal moves. Fixed this by masking the output probability distribution to only legal moves at inference time.

Training Data Scale

Processing 100,000+ games efficiently required batching and pre-tokenizing positions offline rather than on-the-fly, which significantly sped up the training loop.

Compute Constraints

Local training was infeasibly slow. Migrating the training script to Modal with GPU acceleration was the key breakthrough that made iterating on the model practical within the hackathon timeframe.

What I Learned

This project was my first serious end-to-end ML pipeline. I learned how to represent complex game states as tensors, design and train convolutional networks, manage cloud GPU compute with Modal, and debug a model whose outputs aren't immediately human-readable. It also deepened my understanding of how engines like AlphaZero approach chess differently from classical engines.
